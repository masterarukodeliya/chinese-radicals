# Прогресс: переборка тренажёра — слияние форм + 3 уровня

## Решения Ирины
- Все вариации одного ключа — в ОДНОЙ карточке (人/亻, 水/氵/氺, 心/忄 и т.д.). Прогресс пользователя можно сбросить.
- НЕ убирать 香, 鹿, 鼻, 鼠, 麻 (частотные, оставляем).
- Трёхуровневая система СРАЗУ:
  - L1 Обязательные — частые ключи, реально помогают читать совр. иероглифы.
  - L2 Дополнительные — встречаются, но не нужны в самом начале.
  - L3 Справочные/редкие — актуальны как 部首, но в базе лишняя нагрузка.

## Методправки (из разбора ChatGPT, с которыми согласились)
- 厂 → py hǎn (обрыв, склон), не chǎng.
- 王 → main 玉, var 王, mean «нефрит», py yù.
- 月 → «луна/месяц» (убрать «плоть»); добавить отд. карточку 肉 (var 月) «мясо, плоть» ròu.
- 辛 → «острый, едкий; тяжёлый».
- «Неактуальные» → «Справочные / редкие» (L3), не «устаревшие».

## Модель данных (новая)
Карточка: {id (стабильный, для аудио/прогресса), main, variants[], py, mean, level, ex?, expy?, exmean?, mn?}
- Аудио по стабильному id: audio/<id>.mp3 (не по позиции в массиве) — чтобы уровни/порядок не ломали озвучку.
- Очередь: сначала L1, потом L2; L3 только по опту (как нынешние архаичные).
